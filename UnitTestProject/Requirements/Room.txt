Требования к классу Room

# Требования к внешнему пользователю:
    - при создании должен быть выделен уникальный ID (сервис для получения следующего ID?)
    - в здании не может быть двух комнат с одинаковым номером
    - при подписании договора/окончании срока аренды - проверять и обновлять текущий договор
    - в id текущего договора должен быть активный договор (значит, комната занята)
    - комната обязана принадлежать зданию
    - этаж комнаты должен соответствовать кол-ву этажей здания
    - Логика проверки этажа будет вызвана извне при добавлении в здание.
    - в методе создания комнаты нужно получить следующий свободный id для комнаты,
    id здания для которого создается комната, создать комнату с этим id,
    записать этот id в список комнат в здании, сохранить комнату,
    проверить результат сохранения, сохранить здание, проверить результат сохранения
    - комната может быть часть только одного (активного?) договора

# Информация:

ID комнаты в системе [не меняется]
    - строго положительное число

Номер комнаты [не меняется]
    - Только цифры (не больше 4)
    - Одна буква или ее отсутствие (203а)

Площадь комнаты [не меняется]
    - положительное число

Этаж комнаты [не меняется]
    - строго положительное число (упрощение)
    - меньше или равен количеству этажей в здании

Вид отделки [может обновиться]
    - всегда корректен (Enumeration)

Наличие телефона [может обновиться]
    - логический тип данных

Занята ли комната [вычисляемое свойство]
    - логический тип данных

?? ID текущего договора [может обновиться]
    - nullable строго положительное число

ID здания [не меняется]
    - строго положительное число
